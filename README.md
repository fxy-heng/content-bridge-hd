# ContentBridge

ContentBridge 是一个面向创作者的多平台内容发布工具。用户输入一份原始内容后，系统自动适配公众号、知乎、B站、小红书等平台的格式和表达风格，并支持一键发布、发布状态追踪和扩展更多平台的适配器架构；未配置真实账号时自动回退为模拟发布。

## 竞赛题目

题目二：多平台内容发布工具

目标：帮助创作者提升多平台发布效率和便捷性，减少重复改写、手动排版和复制发布的成本。

## 核心功能规划

- 原始内容编辑：标题、正文、标签、封面图。
- 平台适配：公众号、知乎、B站、小红书。
- 风格转换：长文、问答、视频简介、种草笔记等不同表达形式。
- 平台校验：标题长度、标签数量、正文完整性、封面要求。
- 多平台预览：查看各平台适配后的内容。
- 一键发布：后端可用且账号已连接时走真实发布，否则批量模拟发布并展示状态。
- 发布日志：记录平台、标题、状态、时间和失败原因。
- 日历导出：将计划发布时间导出为 `.ics` 文件。
- CSV 报告：导出发布准备度和发布日志，方便复盘。
- 自定义平台：在页面上添加更多平台并自动参与适配。
- 平台预设：导入/导出自定义平台配置，方便复用和分享。
- 规则说明导出：生成平台规则 Markdown，方便评审和维护。
- AI 提示词包：为每个平台生成可复制的 AI 改写提示词。
- 工作区导入导出：保存原始内容、自定义平台、适配结果和发布日志。
- Markdown 导入：从本地 `.md` 草稿快速生成原始内容。
- 草稿版本：保存多个本地版本并恢复。
- 内容模板库：提供工具评测、教程指南、案例复盘等常用创作起点。
- 发布策略建议：根据平台选择和校验结果给出主平台与发布建议。
- 扩展架构：通过平台适配器、平台注册表和发布器接口新增更多平台。

## 当前实现

当前版本是一个无需安装第三方依赖的静态应用：

- Frontend: HTML + CSS + JavaScript ES Modules
- Core: 平台适配器、校验器、模拟发布器
- Test: Node.js 内置断言
- Storage: 浏览器 LocalStorage 保存发布日志

后续如迁移到 React、TypeScript、Vite、后端服务或数据库，需要在 README 和 PR 描述中同步更新依赖说明。

## 原创功能说明

本项目原创部分计划包括：

- 面向不同平台的内容适配规则。
- 平台适配器接口设计。
- 多平台预览和校验工作流。
- 模拟发布状态机和发布日志。
- 自定义平台注册表与通用平台适配器。
- 平台级 AI 改写提示词生成。

## 本地运行

需要 Node.js 18 或更高版本。

### 纯前端模式（模拟发布）

```bash
npm start
```

访问 `http://localhost:5173`。

### 后端模式（真实发布）

真实发布需要启动后端服务：

```bash
cd backend && npm start
```

支持平台：

- **公众号**：需在页面“账号设置”中配置 AppID + AppSecret。真实发布走微信官方 API：获取 access_token、上传封面素材、创建草稿、提交发布。公众号官方接口要求认证公众号、IP 白名单，并且文章必须有封面素材；可填写已有 `thumb_media_id`，也可在内容里提供可公开访问的封面图 URL 让后端自动上传。
- **知乎**：知乎没有稳定公开的文章发布 API，本项目使用 Puppeteer 打开知乎专栏写作页。首次点击“打开知乎登录”，完成网页登录后登录态会保存在 `backend/data/zhihu-profile`，之后“一键发布”会尝试填写标题、正文和话题标签，并给出页面诊断信息。
- **B站**：B站开放平台没有稳定的专栏发布 API，本项目使用 Puppeteer 浏览器自动化打开创作中心。首次点击“打开 B站扫码登录”，扫码后登录态会保存在 `backend/data/bilibili-profile`，之后“一键发布”会尝试填写并提交专栏编辑器。

真实发布不会伪造成功：凭证缺失时回退模拟发布；凭证存在但平台 API 拒绝、封面缺失、登录失效或页面选择器变化时，会在发布日志中标记失败并给出原因。

不配凭证时自动回退到模拟发布模式，不影响现有工作流。
