# Demo 视频脚本

## 1. 开场

说明痛点：创作者经常需要把同一份内容同步发布到公众号、知乎、B站、小红书，但每个平台标题长度、正文结构、标签习惯和封面要求不同，手动适配效率低且容易出错。

## 2. 输入内容

打开 ContentBridge 工作台，展示左侧原始内容编辑区。

操作：

1. 点击“填入示例”。
2. 切换内容模板，说明工具支持工具评测、教程指南、案例复盘等创作起点。
3. 展示标题、正文、标签、目标受众、品牌语气和行动引导。
4. 设置一个未来的计划发布时间。
5. 勾选四个平台。

## 3. 自动适配

点击“自动适配”，展示右侧四个平台的适配结果：

- 公众号：长文结构，有导语、小标题和结语。
- 知乎：观点问答结构，有结论、拆解和建议。
- B站：视频简介结构，有看点和时间线。
- 小红书：短句和话题标签，更适合图文笔记。

同时展示质量评分和平台校验提示。
展示发布准备度看板和发布策略建议，说明哪些平台可直接发布、哪些需要优化。

## 4. 发布前处理

展示每个平台卡片中的能力：

1. 查看平台限制。
2. 点击标题备选，展示标题可替换。
3. 点击“复制文案”。
4. 展开“AI 改写提示词”，说明可接入任意大模型工具。
5. 点击“导出 Markdown”。
6. 点击“导出日历”生成发布提醒。
7. 点击“导出 JSON”生成完整工作区。

## 5. 扩展平台

在“扩展更多平台”中添加一个新平台，例如：

- Key: `douyin`
- 名称：抖音
- 标题上限：30
- 标签上限：6
- 正文建议字数：50
- 平台风格：短视频口播
- 发布类型：短视频文案

添加后展示新平台自动出现在目标平台列表，并能生成适配文案。

## 6. 一键模拟发布

点击“一键模拟发布”，展示发布日志：

- 如果没有设置未来时间，状态为成功。
- 如果设置了未来时间，状态为排期。
- 如果标题或正文为空，状态为失败并显示原因。

## 7. 架构说明

切到 `docs/architecture.md` 或 README，说明平台扩展设计：

- 平台元数据 `platformMeta`
- 平台注册表 `getPlatformRegistry`
- 内置适配器和通用自定义平台适配器
- 校验器 `validateAdaptedContent`
- 发布器 `publishToPlatforms`

强调新增平台只需要新增平台 key、规则、校验和测试，不需要改 UI 主流程。

## 8. 收尾

总结作品完成度：

- 已支持四个平台。
- 已覆盖输入、适配、预览、校验、排期、模拟发布和日志。
- 已提供测试和架构文档。
- 后续可接入真实平台 API 和大模型改写。
