# 使用指南

## 1. 启动

```bash
npm start
```

打开 `http://localhost:5173`。

## 2. 基础发布流程

1. 选择内容模板，或手动输入标题、正文、标签和封面图地址。
2. 填写目标受众、品牌语气和行动引导。
3. 选择计划发布时间。
4. 勾选目标平台。
5. 点击“自动适配”。
6. 在右侧查看各平台版本、质量建议和校验结果。
7. 查看“发布准备度”和“发布策略建议”。
8. 根据需要复制文案、复制 AI 提示词、导出 Markdown 或导出完整 JSON 工作区。
9. 如设置了计划发布时间，可以点击“导出日历”生成 `.ics` 文件。
10. 点击“一键模拟发布”，查看发布日志。

## 3. 内容模板

当前内置三个模板：

- 工具评测：适合评测软件、AI 工具、效率工具。
- 教程指南：适合做步骤型教程。
- 案例复盘：适合发布运营复盘或增长实验。

选择模板后，会自动填入标题、正文、标签、目标受众、品牌语气和行动引导。你可以继续编辑这些内容，再生成平台版本。

## 4. 自定义平台

在“扩展更多平台”中填写：

- 平台 Key：英文、数字、下划线或短横线，例如 `douyin`。
- 平台名称：展示名称，例如 `抖音`。
- 标题上限、标签上限、正文建议字数。
- 平台风格：例如短视频口播、资讯快讯、社区讨论。
- 发布类型：例如短视频文案、图文笔记、动态。
- 是否建议封面。

添加后，该平台会出现在目标平台列表中，并自动参与适配、校验、导出和模拟发布。

## 5. AI 提示词包

每个平台预览卡片都会生成一段“AI 改写提示词”。使用方式：

1. 点击“复制 AI 提示词”。
2. 粘贴到任意大模型工具。
3. 将生成结果复制回平台预览文本框。
4. 再执行复制、导出或模拟发布。

这样即使当前项目不绑定具体 AI 服务，也能进入真实创作者工作流。

## 6. 工作区导入导出

“导出 JSON”会保存：

- 原始内容
- 自定义平台
- 适配结果
- 发布日志

“导入工作区”可以恢复这些内容，适合继续编辑或在不同设备间迁移。

可以参考 `examples/workspace-sample.json` 了解导入文件结构。

如果设置了计划发布时间，可以导出 `.ics` 日历文件，将发布计划加入系统日历或提醒工具。

发布准备度和发布日志也可以导出为 CSV，适合后续复盘、汇报或整理 demo 数据。

## 7. Markdown 草稿导入

点击“导入 Markdown”可以从本地 `.md` 文件读取标题、正文和标签。

推荐格式：

```md
# 标题

tags: AI工具,内容创作

正文内容。
```

## 8. 发布策略建议

发布准备度面板会展示：

- 每个平台是否可发布、需优化或阻塞。
- 当前推荐主平台。
- 针对不同平台的发布建议。

它适合在 demo 中说明项目不是简单改写器，而是覆盖发布前决策的工作台。

## 9. 提交 demo 建议

录制视频时建议展示：

1. 填入示例内容。
2. 自动适配四个内置平台。
3. 添加一个自定义平台，例如抖音。
4. 复制某个平台 AI 提示词。
5. 设置未来计划发布时间并模拟发布。
6. 展示发布日志和架构文档。

## 10. 离线使用

首次通过本地服务访问后，浏览器会缓存核心静态资源。后续在支持 Service Worker 的浏览器中，可以继续打开工作台并访问本地草稿。
