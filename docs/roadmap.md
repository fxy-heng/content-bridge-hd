# 产品路线图

## 当前版本

- 静态前端，无需安装第三方依赖。
- 支持四个内置平台。
- 支持自定义平台扩展，并提供“配置扩展 -> 适配器扩展 -> 发布器扩展”的三层架构蓝图。
- 支持 AI 提示词包、质量评分、发布准备度、排期和模拟发布。
- 支持工作区导入导出。

## 下一阶段：平台扩展框架

- 将高频自定义平台沉淀为内置适配器。
- 为每个平台维护独立规则、提示词、校验和发布边界。
- 为真实发布平台补充统一 Publisher 契约、失败原因和状态回查。
- 将平台预设扩展为可分享的团队规则包。

## 下一阶段：真实 AI 接入

- 增加 AI Provider 配置。
- 将当前提示词包发送给模型服务。
- 支持用户选择“规则适配”或“AI 改写”。
- 保存 AI 改写历史，方便对比不同版本。

## 下一阶段：真实发布接入

- 增加后端服务。
- 增加账号授权和 token 管理。
- 每个平台实现独立 Publisher。
- 发布任务进入队列，支持重试、失败原因和状态回查。
- 按 `docs/api-contract.md` 约定实现发布任务 API。

## 下一阶段：团队协作

- 内容草稿云端保存。
- 支持编辑、审核、发布三种角色。
- 支持 PR 式内容审核记录。
- 支持发布日历和批量排期。

## 下一阶段：商业化增强

- 内容表现数据回收。
- 平台间标题和标签效果对比。
- 生成复盘报告。
- 基于历史表现推荐最佳发布时间和表达风格。
